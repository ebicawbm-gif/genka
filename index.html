このVer0.9の前提（仕様）
・Base（固定）：エイリアス/よみ/基準量/メモ/手入力フラグ など。
・Price（棚卸）：単価/規格/単位 など（CSV取り込みで更新）。
・salesタブは xlsxの並び順を維持。売上順には並べ替えません。
・売価0かつ出数ありのうち、先頭Fは飲み放題メニューとして残します。
・メニューを 複数選択で非表示 できます。
・「〇〇飲み放題」で終わる売上行を合算し、その期間の飲み放題売上として集計します。
・F始まり売価0メニューは飲み放題使用分として原価計算します。
・単品ドリンクは紐づいたレシピの1杯原価 × 出数で計算します。
・飲み放題 / 単品 / 合算 の3パターンで見込み原価と原価率を表示します。

原価マスター（Base×Price → 1g/1ml/1個の原価）

Base（固定）を整えるとレシピ入力が爆速。Price（棚卸）はCSVで更新される想定。
エイリアス空白＝レシピ候補に出さない。入力中にスクロールが飛ばない（行だけ表示更新）。
商品名（key） エイリアス（Base） よみ（Base） 単価(円)（Price） 価格単位（Price） 規格/入数（Price） 基準量（Base） 原価（Base×Price） メモ（Base） 操作
OK = 基準量(Base) > 0 かつ 単価(Price) > 0。
ケース単価なのに規格に24/C/Sが付くことがあるので、基準量(Base)を正に寄せる運用。

レシピ原価（材料×使用量 → 自動原価）

材料は エイリアス / よみ / 商品名 で検索して候補から選ぶ。
売上連携では、出来高を入れると1杯(1人前)原価として使います。
ショートカット：Ctrl + S
入力すると「1杯(1人前)あたり原価」も表示。
